Both SproutOS and Novamira are AI-powered WordPress plugins. Both support PHP execution and follow the same install pattern: add the plugin, activate it, connect your AI client. SproutOS is free with no paid tier. Novamira has a useful free tier but locks Elementor, Bricks, and content management behind a paid plan. The biggest differences are in pricing, per-tool control, and monitoring.

## Pricing

SproutOS has no paid plan. Both the GitHub build and the WordPress.org build are free. PHP execution is only available on the GitHub build due to WordPress.org directory guidelines.

Novamira has three paid tiers:

| Plan | Price | Sites | What's included |
|---|---|---|---|
| Free | €0 | 1 | PHP execution, filesystem tools, sandbox — 10 tools |
| Pro Starter | €49/yr | 3 | Elementor, Bricks, content management, cross-session memory |
| Pro Growth | €99/yr | 10 | Same as Starter, more sites |
| Pro Agency | €249/yr | Unlimited | Same as Starter, unlimited sites |

Novamira's free tier covers PHP execution and filesystem access. Page builder support and content management require a paid plan.

## PHP Execution

Both plugins support PHP execution. The sandbox pattern is the same: AI-written code runs in an isolated directory, and fatal errors disable the file and trigger safe mode automatically.

SproutOS wraps PHP execution with additional safety controls: Safe Mode toggle, per-tool control, and an analytics log. Novamira does not have these layers.

SproutOS requires the GitHub build for PHP execution. The WordPress.org build disables it to comply with directory guidelines.

---

## Page Builder Support

SproutOS includes dedicated ability modules for Elementor and Bricks, both free. They activate automatically when the builder plugin is present on your site. SproutOS also includes 18 cross-builder abilities covering Elementor, Bricks, and Gutenberg.

Novamira adds Elementor and Bricks in its Pro tier. Other builders are not supported natively. PHP execution lets your AI work with unsupported builders through code, but this requires more steps than a dedicated module.

---

## Per-Tool Control

SproutOS lets you enable or disable any of its 175+ abilities individually from the **AI Abilities** tab. You can block file writes while keeping file reads on. You can disable PHP execution without touching anything else. Changes take effect immediately.

Novamira has no per-tool controls. All tools are either fully on or fully off.

---

## Capability Profiles and Token Usage

SproutOS is the only plugin here with Capability Profiles. They control how many abilities are sent to your AI client at session start, which directly affects token usage.

| Profile | Upfront cost | Best for |
|---|---|---|
| Ultra-Minimal | Bridge tools only | Token-sensitive sessions |
| Minimal (default) | ~1,500 tokens / ~6 KB | Most workflows |
| Standard | Moderate catalogue | Builder-heavy sessions |
| Full | Complete catalogue | Maximum tool depth |

All 175+ abilities are reachable in every profile via `sprout-bridge/dispatch-tool`. The profile controls what loads upfront, not what is available.

Novamira sends approximately 14 KB upfront with no way to reduce it.

## Analytics and Monitoring

SproutOS logs every tool call with a timestamp, the WordPress user who triggered it, the ability name, risk level (readonly, additive, destructive, or unknown), and execution time in milliseconds.

The dashboard shows six stat cards: total calls, calls today, failed calls, unique sessions, unique users, and most-used ability. Webhook notifications support Slack, Discord, or any custom endpoint. Payloads are HMAC-signed. CSV export and log purge are also included.

Novamira has no logging or monitoring.

---

## Safety

Both plugins auto-disable a sandbox PHP file that triggers a fatal error and activate safe mode. Neither leaves broken code running.

SproutOS adds additional layers:

- **Safe Mode toggle:** one click makes every ability read-only
- **Protected files:** `.env`, `wp-config.php`, `.htaccess` are hard-blocked
- **Protected directories:** `wp-admin/`, `wp-includes/`, plugin and theme folders cannot be deleted
- **Meta snapshots:** state is captured before destructive changes
- **Per-tool control:** disable any ability without affecting others

Novamira uses sandbox isolation and crash recovery only.

## Frequently Asked Questions

<Expandable title="Is SproutOS really free with no hidden costs?">
  Yes. There is no Pro plan, no usage-based pricing, and no features that require payment. The GitHub build and WordPress.org build are both free. The only difference is PHP execution availability due to WordPress.org guidelines.
</Expandable>

<Expandable title="Does Novamira's free tier cover most workflows?">
  It depends on what you need. If PHP execution and filesystem access are your main use cases, the free tier works well. Elementor, Bricks, and content management all require a paid plan.
</Expandable>

<Expandable title="Does SproutOS have cross-session memory like Novamira Pro?">
  No. SproutOS does not include built-in cross-session memory. Your AI client's own memory and context management handle continuity between sessions.
</Expandable>

<Expandable title="Which plugin uses fewer tokens at session start?">
  SproutOS on the Minimal profile sends approximately 6 KB. The Ultra-Minimal profile sends only the 3 Bridge tools. Novamira sends approximately 14 KB with no way to reduce it.
</Expandable>
