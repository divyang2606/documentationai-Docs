Here's how each piece works together.

## How does the AI connect to my WordPress site?

SproutOS runs as a plugin inside WordPress. Your AI client connects to it directly over HTTPS using your site's URL.

<Steps>
  <Step title="You prompt your AI">
    Type a request in Claude Desktop, Cursor, Windsurf, or any MCP-compatible client. No commands or code needed.
  </Step>

  <Step title="The AI calls SproutOS">
    Your AI sends a tool call to your WordPress site, authenticated with a WordPress Application Password.
  </Step>

  <Step title="SproutOS runs the ability">
    The plugin executes the requested ability inside WordPress using real APIs - `$wpdb`, `WP_Query`, file functions, REST endpoints.
  </Step>

  <Step title="Result returned to your AI">
    Whatever the ability produced - data, file content, confirmation - comes back. Your AI uses it to plan the next step.
  </Step>
</Steps>

No third-party server sits in the middle. No data is proxied through external infrastructure. Your AI client talks to your WordPress site directly.

## What is MCP?

**MCP (Model Context Protocol)** is an open standard for connecting AI clients to external tools. Originally created by Anthropic, it's now adopted across the industry.

Every major AI client already speaks MCP:

- **Claude Desktop** and **Claude Code** by Anthropic
- **Cursor** - AI code editor
- **Windsurf** - AI coding assistant
- **VS Code** and **Zed** - editors with MCP support
- **Cline**, **Continue**, and many more

When you install SproutOS, your WordPress site becomes an MCP server. Any MCP-compatible AI client can connect and use its 175+ abilities, no client-specific code needed.

## What are the 175+ abilities?

**WordPress Abilities API** is a built-in WordPress feature added in WordPress 6.9. It lets plugins register typed, callable tools with defined input and output schemas. SproutOS registers 175+ tools this way, across 10 categories:

| Category | What It Covers |
| --- | --- |
| WordPress Core | Posts, pages, users, media, menus, options, settings |
| File Operations | Read, write, edit, delete files within your site root |
| Theme Management | List, read, edit theme files, update stylesheet |
| PHP Execution | Run PHP inside the WordPress environment (GitHub version) |
| Elementor | Page editing, widgets, global styles, templates |
| WooCommerce | Products, orders, categories, reports |
| Bricks Builder | Full Bricks support, registers when active |
| The Plus Addons | 40+ widget abilities for TPAE |
| Builder | Cross-builder element operations |
| Bridge Tools | Discover any tool, dispatch any ability |

Each ability has a defined input schema (what it accepts) and output schema (what it returns). Your AI reads these schemas so it knows exactly how to call each tool.

## What happens during every task?

Every prompt your AI handles flows through the same four steps:

<Columns cols="2">
  <Card title="1. Discover" icon="search">
    Your AI reads what it needs - plugin source, theme structure, database tables. Only the relevant parts, before writing a single line of code.
  </Card>

  <Card title="2. Prototype" icon="flask">
    Tests code against your actual running WordPress. Iterates, catches errors, adjusts. Real environment, not generic templates.
  </Card>

  <Card title="3. Save" icon="save">
    Working code becomes a sandbox file - loaded on every request, crash-guarded, manageable from the admin dashboard.
  </Card>

  <Card title="4. Expand" icon="rocket">
    Bridge tools let your AI call any of the 175+ abilities on demand. No need to load every tool upfront.
  </Card>
</Columns>

This loop is what makes SproutOS different from a generic MCP server. Your AI works against your real site, not a guess of what your site might look like.

## How is the connection secured?

Every connection requires three things, no exceptions:

- **HTTPS** - all traffic is encrypted in transit
- **WordPress Application Password** - generated per AI client, revocable anytime
- **Administrator role** - only users with `manage_options` can connect

There is no anonymous access. There is no shared secret. Each AI client gets its own credential that you can rotate or revoke from **SproutOS > MCP Connect** at any time.

SproutOS does not phone home, does not proxy through external servers, and does not store credentials outside of WordPress.

<Callout kind="tip">
  Give each AI client its own Application Password. If you ever need to revoke access for one client, the others keep working.
</Callout>

For the full safety model - Safe Mode, Ability Manager, Crash Guard, and more - see [Security & Best Practices](/wordpress-plugin/security).

## Frequently Asked Questions

<Expandable title="Does SproutOS work with any AI client?">
  Any client that supports the Model Context Protocol. This includes Claude Desktop, Claude Code, Cursor, Windsurf, Zed, VS Code with MCP extensions, and many others. The protocol is open, so new clients become compatible automatically.
</Expandable>

<Expandable title="Is my data sent to Anthropic or anyone else?">
  No. SproutOS connects your AI client directly to your WordPress site. There is no third-party server in the middle. Your AI client may send data to its own model provider (Anthropic, OpenAI, etc.), but that's the same as any other AI conversation.
</Expandable>

<Expandable title="Can SproutOS run on shared hosting?">
  Yes. SproutOS is just a WordPress plugin. As long as your host supports WordPress 6.9+ and PHP 8.0+, it works. HTTPS is required for the HTTP transport.
</Expandable>

<Expandable title="What's the difference between SproutOS and a generic MCP server?">
  A generic MCP server gives your AI a few abstract tools. SproutOS gives your AI 175+ real WordPress tools that use real WordPress APIs. Your AI works against your actual site, not a generic interface.
</Expandable>
