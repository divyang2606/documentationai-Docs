MCP stands for **Model Context Protocol** - an open standard that lets AI tools call actions on external systems. SproutOS uses MCP to connect your AI client directly to your WordPress site, so instead of copy-pasting code between windows, the AI reads your files and acts on your site from inside the chat.

Think of MCP as a translator. Your AI speaks MCP. Your WordPress site (with SproutOS installed) listens for MCP. They talk directly - no manual steps in between.

## How MCP Works With SproutOS

SproutOS registers 175+ tools on your WordPress site using the **WordPress Abilities API** - a built-in WordPress feature (WP 6.9+) that lets plugins expose callable actions with defined input and output schemas.

When you ask your AI something, it:
1. Looks at the available tools SproutOS has registered
2. Picks the right one based on what you asked
3. Calls it with the right parameters
4. Returns the result to you in plain language

You never need to know the tool names. The AI figures it out automatically.

**A few examples of what that looks like:**

| What You Say | What the AI Calls |
| --- | --- |
| "Show me my homepage code" | `sprout/read-file` on your front-page template |
| "Change the heading color to blue" | `sprout/update-theme-stylesheet` with the new CSS |
| "Create an About page" | `sprout/create-page` with your title and content |
| "List all my theme files" | `sprout/list-theme-files` on the active theme |

## Why MCP and Not FTP or REST API?

| Method | AI Sees Your Site | Safety Controls | Works With Any AI Client | Setup |
| --- | --- | --- | --- | --- |
| Copy-paste | No | None | Yes | None |
| FTP / SSH | Yes | No sandbox, no restrictions | No | Complex |
| WordPress REST API | Partially | WordPress permissions only | No | Medium |
| MCP via SproutOS | Yes | Sandbox, Safe Mode, Audit Log, per-tool controls | Yes - any MCP client | Simple |

FTP gives the AI raw access with no guardrails. The REST API exposes limited endpoints and requires custom integration per AI client. MCP is designed for AI agents from the ground up - the AI discovers available tools automatically, and SproutOS adds the safety layer on top.

## Which AI Clients Support MCP?

MCP is an open standard. Any client that adopts it can connect to SproutOS with no extra setup:

- **Claude Desktop** - Anthropic's desktop AI application
- **Claude Code** - Anthropic's CLI tool for developers
- **Cursor** - AI-powered code editor
- **Windsurf** - AI coding assistant
- **Zed** - collaborative code editor with MCP support
- **Any future MCP-compatible client** - the protocol is open

## Is MCP Safe?

MCP itself is just a communication protocol - similar to HTTP. The safety comes from what SproutOS adds on top:

- Authentication via WordPress Application Password over HTTPS
- Admin-only access (`manage_options` required)
- Sandbox isolation for all AI-written PHP
- Protected files and directories that can never be touched
- Safe Mode to make everything read-only instantly
- A full Audit Log of every tool call

See [Security & Best Practices](/wordpress-plugin/security) for the full safety model.

## Frequently Asked Questions

<Expandable title="Do I need to understand MCP to use SproutOS?">
  No. MCP works behind the scenes. Install the plugin, connect your AI client, and talk to it in plain language. The AI handles all MCP communication automatically.
</Expandable>

<Expandable title="Does MCP work with ChatGPT?">
  Not natively. As of now, ChatGPT does not support MCP. Claude Desktop, Claude Code, Cursor, Windsurf, and Zed do. Check your AI tool's documentation for MCP support.
</Expandable>

<Expandable title="Is MCP the same as an API?">
  Similar, but designed specifically for AI agents. A traditional API requires the caller to know exact endpoints and formats. MCP lets the AI discover available tools and their parameters automatically - it figures out what to call on its own.
</Expandable>

<Expandable title="Do I need to install anything extra for MCP?">
  Just the SproutOS plugin on your WordPress site, and Node.js 18+ on the machine running your AI client. No extra MCP adapters or proxies needed.
</Expandable>
